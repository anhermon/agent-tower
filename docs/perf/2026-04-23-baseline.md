## Investigation candidates

Routes with First-Load JS **≥ 200 kB** and non-trivial page JS:

1. **`/sessions/[id]`** — 493 kB First-Load, 277 kB page. Outlier by a wide margin. Renders session replay (turn-by-turn transcript with tool calls, code highlighting, markdown, find-in-session search).
2. **`/sessions/overview`** — 257 kB First-Load. Chart-heavy: cost-over-time, cache-efficiency, model-token table, cost-by-project donut.
3. **`/skills`** — 250 kB First-Load. Charts: skills-timeline, skills-heatmap, skills-bar-chart, efficacy dashboard, breakdown stacks.
4. **`/sessions/costs`** — 249 kB First-Load. Charts similar to overview.
5. **`/sessions/activity`** — 247 kB First-Load. Activity heatmap + hour/day breakdowns.
6. **`/sessions/tools`** — 215 kB First-Load. Tool ranking chart + model breakdown donut.

Common hypothesis across 2–6: **Recharts** is being pulled in eagerly as a
client boundary, and every chart-heavy page ships a ~150 kB chart runtime
chunk whether or not the user scrolls to it. `/sessions/[id]` is a
different shape — likely syntax highlighter + markdown renderer + custom
client interactivity.
