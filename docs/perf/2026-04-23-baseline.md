## Confounds worth noting

1. **`controller[kState].transformAlgorithm is not a function` (Node 22 × Next 15 RSC streaming bug).** Fires during server render for some routes. When it hits, Lighthouse sits on `maxWaitForLoad: 60000` before giving up — inflated wall time on a couple of URLs during this sweep by ~6–8 min but did **not** affect the measured metrics we rank on (`navigation` mode measures from browser's POV; timeouts cap load but don't poison TBT/LCP once paint fires). Tracked separately from the perf loop.

2. **Sweep cost.** 14 URLs × 3 runs × ~75 s cold-Chrome per collection ≈ 52 min best case, ~30 min observed. For rapid iteration in Phase 6 we'll use `--collect.url=<one-url>` + `numberOfRuns=1`.

3. **`lighthouse:recommended` preset is overkill for perf gating** — it also runs a11y/SEO/best-practices audits that have separate dedicated gates (`test:a11y`). Dropping to `onlyCategories: ['performance']` cuts collect time ~30%. Leave for next iteration of the CI setup, not this loop.

## Root cause summary for the offender (`/skills`)

Main-thread work (from LHR `mainthread-work-breakdown`):

| Bucket | ms |
|---|---|
| Script Evaluation | **1253** |
| Style & Layout | 285 |
| Other | 42 |
| Rendering | 13 |
| Script Parsing & Compilation | 9 |

Top bootup-time offender: **chunk `7826-f5c92439d13108d8.js`** — 169 KB decoded, **1212 ms eval**, produces 5 long tasks totalling ~1014 ms inside the TBT window (162+187+190+189+286 ms). Same chunk ships on every route, but on green routes TTI is reached at ~500 ms, so its eval falls outside the TBT window.

Eagerly loaded (not lazy in practice): **chunk `952.ce9b099953e49462.js`** — 304 KB decoded, 47% unused. This is the Recharts bundle pulled in by `_lazy.tsx` `dynamic(ssr:false)` imports at hydration time, _before_ the user ever scrolls to the charts.

DOM: **2705 nodes** vs `/kanban` = 133, `/replay` = 157, `/` = 279. `SkillGrid` renders every skill card (≈150) eagerly, no virtualisation, no `content-visibility: auto`. Each card is ~18 DOM nodes. Linear with skill-catalogue size.
